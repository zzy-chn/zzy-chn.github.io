Domjudge 安装与配置

Domjudge 需要安装两部分，一部分是 Domsever，一部分是 judgehost；操作系统为Ubuntu 18.04，domjudge版本为7.1.1。

Domserver安装部分

预先安装的依赖包：

sudo apt-get upgrade && sudo apt-get update
sudo apt install gcc g++ make zip unzip mariadb-server \
        apache2 php php-cli libapache2-mod-php php-zip \
        php-gd php-curl php-mysql php-json php-xml php-intl php-mbstring \
        acl bsdmainutils ntp phpmyadmin python-pygments \
        libcgroup-dev linuxdoc-tools linuxdoc-tools-text \
        groff texlive-latex-recommended texlive-latex-extra \
        texlive-fonts-recommended texlive-lang-european composer
sudo apt install libcurl4-gnutls-dev libjsoncpp-dev libmagic-dev
sudo phpenmod json

编译 Domjudge

wget https://www.domjudge.org/releases/domjudge-7.1.1.tar.gz

这一句是从官网下载domjudge-7.1.1.tar.gz, 但其实更好的办法是先下载下来放到用户家目录，然后直接解压安装

tar -zxvf domjudge-7.1.1.tar.gz
cd domjudge-7.1.1
./configure --prefix=$HOME/domjudge --with-baseurl=127.0.0.1
make domserver && sudo make install-domserver
make judgehost && sudo make install-judgehost
make docs && sudo make install-docs

如果在执行 ./configure --prefix=$HOME/domjudge --with-baseurl=127.0.0.1 这一步时不成功，按照提示在后面添加指定内容即可。

后面三行即安装完毕，分别是安装 domserver、judgehost 和 文档。其中安装 docs时可能会遇到一些 python3 的第三方库安装不完整的提示，需要另外补全安装。

配置数据库

cd ~/domjudge/domserver
sudo bin/dj_setup_database -u root install

配置 Web 服务器

cd ~/domjudge/domserver
sudo ln -s /home/ubuntu/domjudge/domserver/etc/apache.conf \
        /etc/apache2/conf-available/domjudge.conf
sudo a2enmod rewrite
sudo a2enconf domjudge
sudo systemctl reload apache2

第二行中的 ubuntu 是实际的用户名

到这里，就可以访问 http://127.0.0.1/domjudge，并使用 admin 作为用户名和密码登录后台了。密码也有可能生成在

/home/ubuntu/domjudge/domserver/etc/initial_admin_password.secret 中

（如果要别的机器在局域网下面访问，记得开放 80 端口）

配置 MySQL

编辑 /etc/mysql/my.cnf，追加内容

[mysqld]
max_connections = 1000
max_allowed_packet = 16MB
innodb_log_file_size = 48MB

其中 max_allowed_packet 数值改成两倍于题目测试数据文件的大小，innodb_log_file_size 数值改成十倍于题目测试数据文件的大小。

然后重启 mysql

sudo systemctl restart mysql

配置 PHP

编辑 ~/domjudge/domserver/etc/apache.conf，取消以下几行内容前的注释：

<IfModule mod_php7.c>
php_value max_file_uploads      100
php_value upload_max_filesize   128M
php_value post_max_size         128M
php_value memory_limit          512M
</IfModule>

编辑 /etc/php/7.2/apache2/php.ini，搜索 date.timezone 关键字，取消其行前注释，并将其值设为 Asia/Shanghai。

sudo systemctl restart apache2

配置 Apache

编辑 /etc/apache2/apache2.conf，搜索 KeepAlive 关键字，将其值设为 Off，并在其后新增一行内容：

MaxClients 1000

然后重启 Apache

sudo systemctl restart apache2

Judgehost 部分

安装依赖包和功能

sudo apt install make sudo debootstrap libcgroup-dev \
        php-cli php-curl php-json php-xml php-zip procps \
        gcc g++ openjdk-8-jre-headless \
        openjdk-8-jdk ghc fp-compiler \
        libcurl4-gnutls-dev libjsoncpp-dev libmagic-dev \
        supervisor

编译安装

wget https://www.domjudge.org/releases/domjudge-7.1.1.tar.gz

这一步跟上面安装 Domserver 一样，需要下载 Domjudge 。

cd domjudge-7.1.1
./configure --prefix=$HOME/domjudge --with-baseurl=127.0.0.1
make judgehost && sudo make install-judgehost

配置 Judgehost

sudo useradd -d /nonexistent -U -M -s /bin/false domjudge-run

配置 sudoers

将 $HOME/domjudge/judgehost/etc/sudoers-domjudge 复制到 /etc/sudoers.d/ 目录下。

sudo cp $HOME/domjudge/judgehost/etc/sudoers-domjudge /etc/sudoers.d/

修改 rest 密码

使用 vim 等文本编辑器编辑 ~/domjudge/judgehost/etc/restapi.secret 这个文件。把 DOMserver 中相应文件夹下的对应文件抄一下。（如果是多台 judge，一台server，那么配置 judge 时需要抄 server 上面的信息，IP地址也要对应，judge和server的文件对应起来即可）

构建 chroot 环境

使用 vim 等文本编辑器编辑 ~/domjudge/judgehost/bin/dj_make_chroot 脚本，将 ubuntu 镜像改为国内源（第175行）。

# Ubuntu mirror, modify to match closest mirror
[ -z "$DEBMIRROR" ] && DEBMIRROR="http://mirrors.aliyun.com/ubuntu/"

修改之后保存并运行此脚本。这一步会从源上下载必要的软件包，所以请耐心等待。

设置 cgroup

使用 vim 等文本编辑器编辑 /etc/default/grub 这个文件，对其中的这一行做如下修改：

GRUB_CMDLINE_LINUX_DEFAULT="quiet cgroup_enable=memory swapaccount=1"

然后执行：

sudo update-grub

之后重启计算机。

sudo reboot

cat /proc/cmdline

检查是否添加成功。

启动 judgehost

sudo ~/domjudge/judgehost/bin/create_cgroups
~/domjudge/judgehost/bin/judgedaemon

每一次启动都先执行一次 create_cgroups，然后再执行 judgedaemon

公网访问

此时，domjudge已经安装完毕，可以在局域网内进行访问，为了实现公网访问，我们需要将domjudge部署到服务器上，部署的过程和上面一样。

部署完成之后，需要修改以下配置

开启php的错误显示

使用 vim 等文本编辑器编辑 /etc/php/7.2/apache2/php.ini 这个文件，对其中的两行（474行和485行）做如下修改：

display_errors = On
display_startup_errors = On

修改文件权限

将 /home/ubuntu 文件夹极其子文件的权限修改为777，其中 ubuntu 是实际的用户名。

sudo chmod -R 777 /home/ubunut

重新启动apache2服务

sudo systemctl restart apache2

到此为止，环境配置完毕，就可以通过xxx.xxx.xxx.xxx/domjudge访问到主页。

Domjudge 使用

  1. Before Contest
    • Contest 配置比赛
    • Executables 可执行的语言
    • Judgehosts 评测机状态等
    • Judgehost Restrictions 评测机限制
    • Languages 可提交的语言
    • Problems 设置题目
    • Users 用户
    • Teams 队伍
    • Team Categories 队伍分类
    • Team Affiliations 队伍所属机构，即大学
  2. During contest
    • Balloon Status 过题气球状态，送气球时看
    • Clarifications
    • Internal Errors 服务器判题发生的错误等等
    • Rejudgings 重测
    • Scoreboard
    • Statistics/Analysis
    • Submissions
  3. Administrator
    • Configuration settings 通用配置设置
    • Config checker 配置检查，可以看哪个题目没有上传测试数据等等
    • Import / Export 导入导出
    • Manage team passwords 管理队伍密码
    • Refresh scoreboard cache 刷新榜单缓存
    • Judging verifier
    • Audit log 日志

添加队伍和账号

在 Import/Export 中，可以导入队伍和账号

先导入 teams.tsv 再导入 accounts.tst，文件编码要用 UTF-8

  • teams.tsv

第一行为版本行，格式如下：

Field Description Example Type
1 Label teams fixed string (always same value)
2 Version number 1 integer

接下来是队伍描述行，一个队伍占用一行，格式如下：

Field Description Example Type
1 Team Number 3 integer
2 External ID team_003 integer
3 Group ID 3 integer
4 Team name test1_银河之光 string
5 Institution name 长安大学 string
6 Institution short name CHD string
7 Country Code CHN string ISO 3166-1 alpha-3
8 Affiliation External ID 长安大学 string

Group ID 对应 Categories 中的 ID，表示队伍的角色，如女队，打星队伍

  • accounts.tsv

该文件用于描述账号信息，同样包含版本行和账号行，每个账号占用一行。

首行版本行的格式如下：

Field Description Example Type
1 Label accounts fixed string (always same value)
2 Version number 1 integer

账号描述行格式如下：

Field Description Example Type
1 Account Type team string
2 Full Name test1_银河之光 string
3 Username team003 string
4 Password D6rF8l string

Account Type 取值为：team, judge, admin, analyst。这里要导入队伍账号，所以该字段在这固定为 team。

注意：需要设置 Username 的格式才能将账号和队伍关联起来，规则为：Username 的整数部分需要和 team.tsv 中的 Team Number 一致。比如一支队伍 Team Number 为 99，则 Username 可以设置为 team-099。

导入样例：

team.tsv

teams    1                        
3    team003    3    test1_银河之光    长安大学    CHD    CHN    长安大学
4    team004    3    test2_重甲神兵    长安大学    CHD    CHN    长安大学
5    team005    3    test3_雷神之怒    长安大学    CHD    CHN    长安大学
6    team006    3    test4_神风玉堂    长安大学    CHD    CHN    长安大学
7    team007    3    test5_极夜幽冥    长安大学    CHD    CHN    长安大学
8    team008    3    test6_涅槃    长安大学    CHD    CHN    长安大学

accounts.tsv

accounts    1        
team    test1_银河之光    team003    D6rF8l
team    test2_重甲神兵    team004    G6qQ8e
team    test3_雷神之怒    team005    M3xQ4v
team    test4_神风玉堂    team006    A9sF8q
team    test5_极夜幽冥    team007    C8sJ3z
team    test6_涅槃    team008    V7mT8y

tsv文件可以用 Excel 编辑，密码可以用随机函数生成，这里提供一个随机生成6位包含字母数字的函数：

CHAR(INT(RAND()*26+65))&INT(RAND()*9+1)&CHAR(INT(RAND()*26+97))&CHAR(INT(RAND()*26+65))&INT(RAND()*9+1)&CHAR(INT(RAND()*26+97))

另外要注意Excel 保存后的格式一般不是 UTF-8，要用notepad++等工具进行修改

题目导入

迎新赛正式赛的题目已经全部导入

在 Problems 页面下可以编辑 Problem name, Time limit 这些信息，测试数据（Testcases）在页面上只能单点添加和修改，并且不能删除。

如果想批量上传数据，需要按照固定格式打包上传

.
├── data
│   ├── sample
│   └── secret
├── submissions
│   ├── accepted
│   ├── time_limit_exceeded
│   └── wrong_answer
├── domjudge-problem.ini
├── problem.pdf
└── problem.yaml

如上为一般题目的文件格式

https://www.domjudge.org/docs/manual/7.3/problem-format.html

data文件夹中存放样例和测试数据（如果提供样例，那么比赛时用户可以进行下载）

输入文件以 .in 结尾，输出文件以 .ans 结尾，不论是否为SPJ都需要提供 .ans 文件。不论是sample还是secret，最终都会被评测机进行测试

submissions 文件夹不是必须提供的，但是提前准备好 AC、WA、TLE等代码可以方便进行测试，这是准备一场完善的比赛所需要考虑到的

domjudge-problem.ini 包含了一些键值对信息，例如：

probid = G
name = 咖啡原料
timelimit = 1
color = #E6E6FF

problem.pdf 为该题目的题面

problem.yaml

limits:
  memory: 64

该题目的内容准备好之后，将这几个文件夹打包 zip 格式压缩，进行上传（注意是直接将这几个文件夹进行打包上传，不需要再把他们装入一个新的文件夹在进行打包）压缩后的名称会成为该题目的简写。

上传新题目时，可以在此处上传，要先选择对应的比赛

也可以对某个题目进行修改

如果题目需要用到SPJ，那么需要使用 testlib 编写SPJ程序，所需要的 testlib.h 也需要打包上传

由于 domjudge 对于 checker 的入口参数和返回值与传统的 checker 较为不同，需要使用 DomJudge 适配版本的 testlib.h。

下载地址：https://github.com/cn-xcpc-tools/testlib-for-domjudge

将编写的 cheker.cpp 以及 testlib.h 放入 output_validators/{VALIDATOR_NAME} 中

在 problem.yaml 中添加一行 validation: custom

格式如下：

.
├── data
│   ├── sample
│   └── secret
├── output_validators
│   └── validate
│       ├── checker.cpp
│       └── testlib.h
├── submissions
│   ├── accepted
│   ├── time_limit_exceeded
│   └── wrong_answer
├── domjudge-problem.ini
├── problem.pdf
└── problem.yaml

其他

  • 目前两台机器的账号都是 chd-acm， 密码为 CHD-ACM

  • 如果比赛（正式赛或热身赛）题面有更改，要在A题的修改页面重新上传 problem_txt
